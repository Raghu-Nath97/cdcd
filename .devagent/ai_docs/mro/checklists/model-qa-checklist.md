# Model QA Checklist

**Source:** [Confluence - Model QA Checklist](https://jira-pg-ds.atlassian.net/wiki/spaces/HWR/pages/358351008/Model+QA+Checklist)

## Purpose

This checklist shall ensure quality models are used that are fit for the business case and follow best practices. This document is owned by the Algorithm Owner, and to be signed off by the Model Risk Officer. Update with any major change in the model.

---

## Quick Assessment Questions

As generic questions, starting off a review, the following items can be answered instead of a complete review with the entire table:

1. Does COVID-19 pandemic impact your models?
2. Are goals, measures, assumptions & objectives clearly defined per business objectives?
3. How do you define & measure the error function?
4. Are missing data and outliers being handled per business use case or model requirements?
5. Have you checked autocorrelation / partial autocorrelation?
6. Are features robust over time and other dimensions?
7. Are you using smallest possible number of features? How does the number of dimensions affect the model?
8. Are algorithm choices per target or goal defined?
9. Is the model planned to refit/rebuild periodically? If so, a refit/rebuild plan should be drafted.
10. How were the hyperparameters determined?
11. Is selection of features and hyperparameters based on cross-validation? If so, are they optimized in the same cross-validation loop?
12. What steps were taken to ensure minimization of overfitting or underfitting?
13. Are validation errors and training errors derived from the same distribution?
14. Do features coefficients or importance make sense?
15. Has the target been checked for partial dependence on features?

### Model Basics - Target/Partitioning/Metrics

#### Model Target/Goal

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Is your modeling target or goal (dependent variable, effect, cost function) defined per business need? Is it defined by the business, or generated by the model? | E.g., y = f(sales), number of clusters in case of unsupervised learning, classes, etc. | |

#### Data Partitioning

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Do you need to partition the data? If so, how did you partition it, and was it based on business case and model requirements? | Avoid info leakage when normalizing, percentiling, PCA, SVD. Partitioning into test/valid/train sets prior to feature transformation | |
| Are classes balanced? Did you compensate unbalanced classes? | | |
| Did you randomize and/or stratify? | | |
| If time series / geospatial, are data explicitly partitioned? | E.g. do not separate data from individuals when modelling individual behaviour | |
| Are different partitions of data weighed differently? If so, what is the basis of weights? | E.g., if time series data, recent period are weighed more than older periods | |
| How do weights impact training and model performance & predictability? | | |
| Are weights defined by a function or model, tuned by model, or user-defined? | | |

#### Model Metrics

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| How do you define & measure the error function? Are you over penalizing outliers, e.g., with MSE or classification errors? | E.g. MAPE, MAE, etc. | |
| Is your target (modeled value) a function of order-of-magnitudes? Is it scale-dependent? | | |
| Which tests are used to ensure it does not model noise? | E.g. Target Shuffling, Boruta | |

### Features

#### Feature Correction

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Are missing data (dropping observations or filling) being handled per business use case or model requirements? Is this documented? | NaN, NA, 0, etc. | |
| How are outliers being defined and subsequently handled (e.g., exclusions?) Were reasons of exclusion / inclusion documented? | E.g. percentile, IQR handling - impute, exclude, etc. | |
| How are extreme or anomalous data being handled? | E.g., data in tails of distribution, or data in peaks | |

#### Feature Transformation

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| If features being transformed, what's the rationale? | Log, percentile, image rotation, cropping, hot encoding, etc. | |
| Is transformation reversible? Is any information lost in conversion? | | |
| For transformed features - what is the Variability Change before/after? | E.g. dynamic range/contrast | |

#### Feature Augmentation

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Time series Models: Is there a seasonality variable, as needed? Have you checked autocorrelation / partial autocorrelation? | Days of week, quarter, trend variable, holidays | |
| Have features been created from encoded features? If so, decoding is static or dynamic? | | |
| Are features robust over time and other dimensions? | Splitting, incremental features, added income attributes, etc. | |

#### Feature Reduction

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Was any feature reduction implemented? Why? How does it impact model results and performance, explainability etc? | PCA, SVD, combining features, Lasso, Bootstrap, etc. | |

#### Feature Assertion

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| How much did the data volume change upon engineering, and what is the impact? | Observations and/or features gained or lost | |
| Have descriptive features been calculated? Did you use any known rules or intended modelling and why? How do they compare across use cases and over time? | Document business rules / assumptions | |
| Has a feature assertion table been used to assess feature value distribution, and any changes? | Category or brand annualized sales, changes over time, Ranges, distinct values, Moments, mean, SD, skewness, Data density? | |
| Could it be that the data is too good to be true? | | |

### Modeling

#### Algorithms

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Are algorithm choices per target or goal defined? Does the selected algorithm have bias toward or against a specific type of prediction? | E.g., linear model vs. trees for feature importance goals | |
| Is algorithm type recommended for type of model desired for a business problem - predictability vs. explainability requirements? Reliability and dependability? | | |
| Is the model dependent on (an)other model(s)? If so, is there a remediation plan if the other model(s) fail, or retired? | | |
| Is the model planned to refit/rebuild periodically? If so, a refit/rebuild plan should be drafted. | | |
| Do you have sufficient and necessary data for training selected algorithm? | E.g., Learning curve flattens with sufficient training data | |
| As you defined the approaches - what is your simple model you tried to beat, and how does the performance compare? What are the bottom & upper limits of the model performance to expect? | E.g., comparison of results across different ml algorithms used (such as, random forest vs logistic regression vs deep learning) | |

#### Hyperparameters, Model Selection, and Overfitting

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| How were the hyperparameters determined? | Search space, test procedure, Bayesian optimization | |
| Is cross-validation used for modeling? If so, how many folds vs. data size? | | |
| Is selection of features and hyperparameters based on cross-validation? If so, are they optimized in the same cross-validation loop? | | |
| How many times cross-validation is repeated? | | |
| Do training and validation folds have similar variance? | | |
| Are folds randomly selected or stratified folds are used? | | |
| What steps were taken to ensure minimization of overfitting or underfitting? Are number of features more than number of observations? | | |
| Are validation errors and training errors derived from the same distribution? | | |
| What metrics are used for features, hyperparameters, and model selection? | | |
| Is the model validated against a holdout (out-of-bag) sample? If so, are holdout errors (or MAPE) and training errors (or MAPE) similar size, and have similar variance? | | |

#### Model Interpretation

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Do features coefficients or importance make sense? | | |
| Does your model explain (or you can explain) reduced accuracy, precision, recall, or other measures of model performance? If not, can it be manually explained? What conclusions can you draw about your model from explanations? | E.g., are you able to explain misclassified records (in classification ml project) manually? Does you model explain results better than naive scenario | |
| Has the target been checked for partial dependence on features? | | |
| Has sub-population analysis been done on model metrics? Are model metrics similar across different quantiles/deciles of feature data set? Are there any sub-population biases? | E.g., check biases for certain demographics | |

#### Model Performance

| Question | Examples & Details | Answers, Links & Notes |
| --- | --- | --- |
| Is your overall model a combination (serial or parallel) of multiple sub-models? If so, how do errors from stacking sub-models add up? | Use ceiling analysis | |
| If using modeled data, how do errors propagate? How does performance of one sub-model affect that of the other models in the stack? | | |
| Can you identify and evaluate "weakest link in the chain"? | | |
| Are residuals checked to find potential problems? If a time series model, is there a serial correlation in residuals? | | |

| Is the full taxonomy of the model described? | | |
| Is model performance being monitored across different scenarios? | | |
